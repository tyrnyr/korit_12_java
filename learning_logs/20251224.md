# Method
## 정의
Java에서는 함수(function)의 개념과 동일함. JavaScript / Python에서는 모든 함수가 클래스에 종속되어있기 때문에 method로 통일.

- function vs. method
그렇다면 용어를 왜 분리하는가?
   - 기본적으로 클래스 내부에 함수를 정의하게 되면 이를 '매서드'라고 합니다. Java는 모든 함수들이 특정 클래스 내부에 있습니다.(저희가.java 파일을 생성할 때 클래스를 만든다고 하죠, 그리고 그 내부에 method들을 정의하게되기 때문에 클래스 내부에 함수가 있다고 해석할 수 있씁니다.)
   그래서 다른 언어를 배우게 된다면 '함수의 호출 방식'과 '메서드의 호출방식' 상에서의 차이가 존재하지만, 현재 저희 수업을 기준으로 했을때는 전부 다 method에 해당하고, 제가 Java 수업중에 함수 어쩌고저쩌고 하면 그냥 method라고 알아들으시면 되겠습니다.(타 언어 수업 때는 좀더 엄밀히 구분하겠습니다.)
   ## 용어 정리
   1. method 정의(define) : 사용자가  메서드를 새로 만드는 것을 의미
   2. method 호출(call) : 만들어진(혹은 미리 만들어진) 메서드를 실행시키는 것을 의미
   3. 매개변수(parameter) : 특정 메서드에 필수적으로 요구되어지는 변수 정의 시에 요구됨.
   4. 출력값 / 결과값 / 리턴값(return) : 메서드의 호출 결과 생성되는 자료(없을 수도 있음 / output 에 해당)
   5. 인수 / 인자(argument) : 사용자가 메서드에 전달하는 입력 호출시에 요구됨.
## Java 상에서의 method
예를 들어, '사진을 찍는다'라는 행위에 대해서 설명하자면
1. 주머니에서 폰을 꺼내고,
2. 잠금 화면을 풀고(그런데 폰마다 Face ID인 경우가 있고 패턴인 경우도 있고 etc)
3. 카메라를 켜고
4. 사진을 찍고자 하는 대상에 폰을 조준하고,
5. 셔터를 누른다.
라는 해당 과정이 존재합니다. 그런데 컴퓨터는 생각보다 너무 멍청하기 때문에 '사진을 찍는다'라는 명령을 내리는 것으로는 충분하지 않고, 1~5까지의 명령어를 하나하나 '순서대로' 내려야 비로소 '사진을 찍는 행위'가 완료되게 됩니다.

그런데 사진 한 장 찍고 싶을 때마다 1-5를 매번 코드를 입력한다고 하면 너무 귀찮을 겁니다. 그래서 1-5에 해당하는 명령들을 순서대로 묶어놓는 행위를 미리해두는 방법을 사용하는데 이를 'method 정의'라고 하게 됩니다.

그리고 실제로 컴퓨터에게 '사진을 찍는다'라는 명령을 내리게 되는것을 'method 호출'이라고 표현할 수 있겠씁니다.

여러분의 폰이 아이폰인지 안드로이드폰인지에 따라서 잠금화면을 푸는 방법도 다르고, 사진을 찍는 방식도 다를겁니다. 그렇다면 특정 폰을 가지고 있음을 컴퓨터에게 알릴 필요가 있을건데, 이를 argument라고 할 수 있겠습니다. 그리고 사진을 찍으면 사진 결과물이 남을텐데, 이를 return이라고 할 수 있겠습니다.
